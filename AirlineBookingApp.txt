✅ Key Notes about this structure:

Backend

controllers → handles API logic

middleware → auth, error handling, role-based access

models → Mongoose models for User, Flight, Booking

routes → API endpoints

utils → helper functions (e.g., JWT token generation, input validation)

config/db.js → MongoDB connection

server.js → main Express server entry point

Frontend

app/ → main Angular app folder

auth/ → contains login and signup folders with .html, .scss, .ts, .spec.ts

flights/ → flight-list and flight-add components

bookings/ → booking-list and booking-add components

services/ → handles API communication

app.routes.ts → all routing configurations

app.ts → root standalone Angular component

styles.scss → global styling

environments/ → environment configs

Spec.ts files are included for unit testing for each component.
